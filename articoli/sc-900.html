Microsoft SC-900 Security Identity Compliance Certificazione

Il 9 maggio sostengo l'esame SC-900, Microsoft Security, Compliance and Identity Fundamentals. Ho costruito questo sistema per non limitarmi a prendere appunti ma per ritenere davvero quello che studio. E lo pubblico perche' potrebbe essere utile anche a te.

L'esame richiede un punteggio minimo di 700 su 1000. Non chiede di saper configurare nulla: chiede di capire cosa fa ogni strumento Microsoft nell'ecosistema di sicurezza cloud, e perche' esiste. La distinzione e' importante, perche' cambia completamente il modo in cui si studia.

Non memorizzare. Costruisci una mappa mentale di come i prodotti Microsoft si relazionano tra loro. L'esame ragiona sempre per scenari.

I 5 principi che reggono tutto

Queste non sono definizioni da memorizzare. Sono le lenti attraverso cui leggere ogni domanda dell'esame.

🏰
Difesa in profondita' -- 7 strati
La sicurezza non e' un muro unico ma una serie di strati sovrapposti. Ogni strato assume che il precedente possa fallire: Fisica -- Identita' -- Perimetro -- Rete -- Calcolo -- Applicazione -- Dati.
🔐
Zero Trust -- tre pilastri
Verifica esplicita: autenticare sempre basandosi su tutti i segnali disponibili. Minimo privilegio: accesso Just-In-Time e Just-Enough-Access. Assumi la violazione: segmenta, cripta, minimizza il raggio d'esplosione.
🪪
L'identita' e' il nuovo perimetro
In un mondo cloud non esiste un confine fisico da difendere. L'identita' -- utente, dispositivo, applicazione -- e' l'unico controllo perimetrale rimasto. Il cliente e' SEMPRE responsabile di identita' e accessi, in qualsiasi modello cloud.
⚖️
Responsabilita' condivisa
Microsoft gestisce l'infrastruttura fisica. Il cliente gestisce dati, identita' e accessi sempre -- in IaaS, PaaS e SaaS. Il confine si sposta, ma la responsabilita' sui dati non si sposta mai.
🎛️
Autenticazione prima, autorizzazione dopo
Autenticazione: sei chi dici di essere? Risponde alla domanda CHI. Accade prima. Autorizzazione: hai il permesso di fare questo? Risponde alla domanda COSA. Accade dopo. Mai invertire l'ordine.
Sicurezza = Prevenzione + Rilevamento + Risposta
Zero Trust = Verifica esplicita + Minimo privilegio + Assumi la violazione
Identita' = Autenticazione (chi) + Autorizzazione (cosa) + Governance (quanto)

Mappa prodotto in 60 secondi

Se sai collocare ogni prodotto in questa mappa, sei pronto per l'esame.

Identita' -- Entra
Entra ID = directory cloud (ex Azure AD)
MFA = verifica identita' (3 fattori)
Accesso condizionale = regole IF-THEN
PIM = accesso privilegiato Just-in-Time
RBAC = permessi per ruolo
ID Protection = rilevamento rischio account
Sicurezza -- Defender + Azure
Defender XDR = piattaforma unificata
Sentinel = SIEM + SOAR cloud-native
Defender for Cloud = postura CSPM
Key Vault = segreti, chiavi, certificati
Azure Firewall = traffico L3-L7
WAF = protezione app web (OWASP)
Conformita' -- Purview
Compliance Manager = punteggio conformita' DLP = blocca fuga di dati Etichette riservatezza = classifica e protegge eDiscovery = indagini legali Insider risk = minacce interne Audit = tracciamento attivita'
